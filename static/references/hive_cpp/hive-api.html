ElaLogLevel_Trace = 6¶

Client instance functions¶

hive_client_new¶

HiveClient *hive_client_new(const HiveOptions *options)¶

Create a new hive client instance to the specific backend.

If the data is stored under options.peristent_location, the client instance would be created base on it. Otherwise, a new client instance would be created and returned without any user context. On sucessfully returning, the state of client instance is “raw”.

All other hive APIs should be called after having client instance.

Return
If no error occurs, return the pointer of Hive client instance. Otherwise, return NULL, and a specific error code can be retrieved by calling hive_get_error().
Parameters
  • options: [in] A pointer to a valid Options structure of specific backend.

hive_cient_close¶

int hive_client_close(HiveClient *client)¶

Close the hive client instance.

After closing client instance, all derived instances of this client, such as drive and file instances would become invalid. And any calling APIs with invalid drive and file instance would be undefined.

All derived instances such as drive and files should be closed before calling this API.

Return
If no error occurs, return 0. Otherwise, return -1, and a specific error code can be retrieved by calling hive_get_error().
Parameters
  • client: [in] A handle identifying the Hive client instance.

hive_client_login¶

int hive_client_login(HiveClient *client, HiveRequestAuthenticationCallback *callback, void *context)¶

Associate a user with the client.

During the process, the user delegates to the client. On successfully returning, the state of client becomes “logined”.

This function is effective only when state of client is “raw”.

Return
If no error occurs, return 0. Otherwise, return -1, and a specific error code can be retrieved by calling hive_get_error().
Parameters
  • client: [in] A handle identifying the Hive client instance.
  • callback: [in] Callback to open authentication URL during login process.
  • context: [in] User data to be passed as context parameter to callback.

hive_client_logout¶

int hive_client_logout(HiveClient *client)¶

Dissociate the user from the client.

All client’s data in persistent location is deleted. All derived instances of this client, such as drive and file instances would become invalid. And any calling APIs with invalid drive and file instance would be undefined.

This function is effective only when state of client is “logined”. client.

Return
If no error occurs, return 0. Otherwise, return -1, and a specific error code can be retrieved by calling hive_get_error().
Parameters
  • client: [in] A handle identifying the Hive client instance.

void ela_log_init(ElaLogLevel level, const char *log_file, void (*log_printer)(const char *format, va_list args))¶

Initialize log options for Hive.

If this API is never called, the default log level is ‘Info’; The default log file is stdout.

  • log_file: [in] the log file name. If the log_file is NULL, Hive will not write log to file.
  • log_printer: [in] the user defined log printer. Can be NULL.

hive_get_error¶

int hive_get_error(void)¶

Retrieves the last-error code value.

The last-error code is maintained on a per-instance basis. Multiple instance do not overwrite each other’s last-error code.

Return
The return value is the last-error code.
